创建执行计划（Plan）。当用户任务较复杂、涉及多个可独立执行的步骤时，使用本工具生成结构化计划。计划将在执行前提交用户审批。

### 要求
- 步骤数量 3-10 步为宜
- 每个步骤应可独立验证，标题简洁、描述包含具体操作和预期产出
- 参数必须是**单个合法的 JSON 对象**，不要使用 Markdown 代码块包裹，不要附加解释文字
- 字符串中的双引号必须转义：`\"`，使用 `\n` 表示换行

### 示例
{"title":"项目部署","steps":[{"title":"环境检查","description":"检查服务器配置和依赖是否满足要求"},{"title":"代码拉取","description":"从仓库拉取最新代码到部署目录"},{"title":"构建打包","description":"执行构建命令，生成可部署产物"},{"title":"服务重启","description":"重启应用服务并验证健康状态"}]}

### 1.2 核心设计原则

1. **统一接口**：所有厂商的 ChatModel 都实现 Spring AI 的 `ChatModel` 接口，`AgentRuntime` 无需关心厂商细节
2. **工厂模式**：`ChatModelFactory` 根据 provider type 创建对应的 ChatModel 实例
3. **注册表缓存**：`ChatModelRegistry` 按 provider 维度缓存 ChatModel，避免重复创建
4. **热重载**：厂商配置变更时清除对应缓存，下次请求时自动重建
5. **向后兼容**：保留 DashScope 自动配置作为 fallback

| 阶段 | model 字段内容 | 解析方式 |

### 3.2 各厂商 ChatModel 创建方式

#### DashScope

```java
import com.alibaba.cloud.ai.dashscope.api.DashScopeApi;
import com.alibaba.cloud.ai.dashscope.chat.DashScopeChatModel;
import com.alibaba.cloud.ai.dashscope.chat.DashScopeChatOptions;

public ChatModel createDashScope(String apiKey, String baseUrl) {
    DashScopeApi api = baseUrl != null
        ? new DashScopeApi(baseUrl, apiKey)
        : new DashScopeApi(apiKey);

    DashScopeChatOptions options = DashScopeChatOptions.builder().build();
    return new DashScopeChatModel(api, options);
}
```

#### OpenAI Compatible

```java
import org.springframework.ai.openai.OpenAiChatModel;
```

### 3.3 per-request 模型覆盖

各厂商的 ChatModel 都支持 per-request 设置模型名，通过 `ChatOptions`：

```java
// DashScope
DashScopeChatOptions.builder().withModel("qwen-max").build();

// OpenAI Compatible
OpenAiChatOptions.builder().model("gpt-4o").build();

// Anthropic
AnthropicChatOptions.builder().model("claude-3-5-sonnet-20241022").build();
```

将 `ChatOptions` 合并到 `Prompt` 中即可覆盖默认模型：

```java
ToolCallingChatOptions chatOptions = ToolCallingChatOptions.builder()
    .toolCallbacks(tools)
    .internalToolExecutionEnabled(false)
    .model(resolvedModelId)  // per-request 模型覆盖
    .build();
Prompt prompt = new Prompt(history, chatOptions);
chatModel.stream(prompt);
```

### 4.6 ChatModelRegistry

```java
package com.atm.javaclaw.agent.model;

import org.springframework.ai.chat.model.ChatModel;
import org.springframework.stereotype.Component;

import java.util.concurrent.ConcurrentHashMap;

/**
 * ChatModel 注册表。
 * - 按 provider 维度缓存 ChatModel 实例
 * - 支持通过 model_definition.id 解析到具体的 ChatModel + modelId
 * - 厂商配置变更时调用 evict() 清除缓存
 */
@Component
public class ChatModelRegistry {

    private final ChatModelFactory factory;
    private final ConcurrentHashMap<Long, ChatModel> providerCache = new ConcurrentHashMap<>();
    private final ConcurrentHashMap<Long, ModelConfig> modelConfigCache = new ConcurrentHashMap<>();
    private final ConcurrentHashMap<Long, ProviderConfig> providerConfigCache = new ConcurrentHashMap<>();

    // fallback：Spring AI 自动配置的默认 DashScope ChatModel
    private final ChatModel defaultChatModel;

    public ChatModelRegistry(ChatModelFactory factory, ChatModel defaultChatModel) {
        this.factory = factory;
        this.defaultChatModel = defaultChatModel;
    }

    /**
     * 根据 model_definition.id 解析出可用的 ChatModel 和 modelId。
     *
     * @param modelDefinitionId model_definition.id 或旧格式的模型名称
     * @return 解析后的模型，找不到时返回默认模型
     */
    public ResolvedModel resolve(String modelDefinitionId) {
        // 1. 尝试数字解析 → model_definition.id
        //    → 查 modelConfigCache → 拿 providerId
        //    → 查 providerCache → 拿 ChatModel
        //    → 返回 ResolvedModel(chatModel, modelId, ...)

        // 2. 非数字 → 旧格式兼容（如 "qwen-max"）
        //    → 在默认 DashScope provider 下查找
        //    → 返回 ResolvedModel(defaultChatModel, modelName, DASHSCOPE, ...)

        // 3. 都找不到 → 返回 fallback
        //    → ResolvedModel(defaultChatModel, modelDefinitionId, DASHSCOPE, "default")
    }

    /**
     * 注册或更新一个 provider 的配置。
     * 创建 ChatModel 实例并缓存。
     */
    public void registerProvider(ProviderConfig config) {
        ChatModel chatModel = factory.create(config);
        providerCache.put(config.id(), chatModel);
        providerConfigCache.put(config.id(), config);
    }

    /**
     * 注册模型定义（用于 resolve 映射）。
     */
    public void registerModel(ModelConfig config) {
        modelConfigCache.put(config.id(), config);
    }

    /**
     * 清除某个 provider 的缓存（配置变更时调用）。
     */
    public void evictProvider(Long providerId) {
        providerCache.remove(providerId);
        providerConfigCache.remove(providerId);
    }

    /**
     * 全量刷新（从 DB 加载所有 provider 和 model 配置）。
     * 在应用启动时和厂商 CRUD 后调用。
     */
    public void refreshAll(
            java.util.List<ProviderConfig> providers,
            java.util.List<ModelConfig> models) {
        providerCache.clear();
        providerConfigCache.clear();
        modelConfigCache.clear();

        for (ProviderConfig p : providers) {
            registerProvider(p);
        }
        for (ModelConfig m : models) {
            registerModel(m);
        }
    }
}
```

```java
@Service
public class AgentRuntime {
    private final ChatModelRegistry chatModelRegistry;  // 替换 ChatModel
    private final ToolsEngine toolsEngine;
    private final RunQueueManager runQueueManager;
    private final ObjectMapper objectMapper;

    public AgentRuntime(ChatModelRegistry chatModelRegistry,
                        ToolsEngine toolsEngine,
                        RunQueueManager runQueueManager,
                        ObjectMapper objectMapper) {
        this.chatModelRegistry = chatModelRegistry;
        // ...
    }

    private Flux<AgentEvent> executeAgentLoop(AgentRunRequest request) {
        String systemPrompt = buildSystemPrompt(request.agent());
        ToolCallback[] tools = toolsEngine.getToolCallbacksFor(
            request.toolsEnabled(), request.mcpToolsEnabled());
        int maxTurns = request.agent().getMaxTurns();
        Duration timeout = Duration.ofSeconds(request.agent().getTimeoutSeconds());

        // 新增：根据 agent 配置解析模型
        ResolvedModel resolved = chatModelRegistry.resolve(request.agent().getModel());

        log.info("Agent '{}' model: {} (provider={}, type={}), tools: {} total",
                request.agent().getName(),
                resolved.modelId(), resolved.providerName(), resolved.type(),
                tools.length);

        // 改造：设置 per-request model
        ToolCallingChatOptions chatOptions = ToolCallingChatOptions.builder()
                .toolCallbacks(tools)
                .internalToolExecutionEnabled(false)
                .model(resolved.modelId())  // 新增：per-request 模型覆盖
                .build();

        // 改造：使用解析后的 ChatModel
        return executeLoopTurn(
            conversationHistory, chatOptions, maxTurns, timeout, 1,
            new StringBuilder(), resolved.chatModel());  // 传入对应的 ChatModel
    }

    private Flux<AgentEvent> executeLoopTurn(
            List<Message> history,
            ToolCallingChatOptions options,
            int maxTurns,
            Duration timeout,
            int turn,
            StringBuilder fullText,
            ChatModel chatModel) {  // 新增参数
        // ...
        Prompt prompt = new Prompt(new ArrayList<>(history), options);
        Flux<AgentEvent> streaming = chatModel.stream(prompt)  // 使用传入的 chatModel
                .timeout(timeout)
                // ...
    }
}
```

### 5.3 关键变化点

| 改造点 | 原代码 | 新代码 |
|--------|--------|--------|
| 注入依赖 | `ChatModel chatModel` | `ChatModelRegistry chatModelRegistry` |
| 模型解析 | 无 | `chatModelRegistry.resolve(agent.getModel())` |
| ChatOptions | 无 model 设置 | `.model(resolved.modelId())` |
| 模型实例 | 全局 `this.chatModel` | `resolved.chatModel()` |

### 6.4 ModelRegistryService（DB ↔ Registry 桥接）

```java
@Service
public class ModelRegistryService {
    private final ModelProviderRepository providerRepo;
    private final ModelDefinitionRepository modelRepo;
    private final ChatModelRegistry registry;
    private final CryptoService crypto;

    /**
     * 从 DB 加载所有 enabled 的厂商和模型，注册到 ChatModelRegistry。
     * 在 ApplicationReadyEvent 和厂商 CRUD 后调用。
     */
    public Mono<Void> refreshRegistry() {
        return providerRepo.findAllByEnabledOrderBySortOrder(1)
            .flatMap(entity -> {
                ProviderConfig config = toProviderConfig(entity);
                return modelRepo.findByProviderIdAndEnabledOrderBySortOrder(entity.getId(), 1)
                    .map(this::toModelConfig)
                    .collectList()
                    .map(models -> Map.entry(config, models));
            })
            .collectList()
            .doOnNext(entries -> {
                List<ProviderConfig> providers = entries.stream()
                    .map(Map.Entry::getKey).toList();
                List<ModelConfig> models = entries.stream()
                    .flatMap(e -> e.getValue().stream()).toList();
                registry.refreshAll(providers, models);
            })
            .then();
    }

    private ProviderConfig toProviderConfig(ModelProviderEntity entity) {
        return new ProviderConfig(
            entity.getId(),
            entity.getName(),
            ProviderType.fromString(entity.getType()),
            entity.getBaseUrl(),
            crypto.decrypt(entity.getApiKeyEncrypted())
        );
    }

    @EventListener(ApplicationReadyEvent.class)
    public void onStartup() {
        refreshRegistry().subscribe();
    }
}
```

## 10. 迁移方案

### 10.1 数据迁移流程

```
Step 1: Flyway V7 → 创建 model_provider 和 model_definition 表
    │
Step 2: Flyway V8 → 插入默认 DashScope 厂商和模型（api_key 占位符）
    │
Step 3: ApplicationReadyEvent
    │  ModelRegistryService.onStartup()
    │    ├── 检查默认 DashScope 厂商的 api_key 是否为占位符
    │    │   是 → 从 spring.ai.dashscope.api-key 读取
    │    │        → 加密 → 更新 DB
    │    ├── 检查 agent.model 字段格式
    │    │   旧格式("qwen-max") → 查找 model_definition
    │    │                       → 更新为 id
    │    └── 全量加载 providers + models → 注册到 Registry
    │
Step 4: 系统正常运行，Registry 已就绪
```

### 10.2 兼容性保证

| 场景 | 处理方式 |
|------|---------|
| agent.model 为旧字符串格式 | ChatModelRegistry.resolve() 按名称在默认 DashScope provider 下查找 |
| agent.model 为空 | 使用默认 DashScope 模型 |
| agent.model 为不存在的 id | 回退到默认 DashScope ChatModel + agent.model 作为模型名 |
| model_provider 表为空 | 从 application.yml 自动创建默认厂商 |

## 12. 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Spring AI starter 冲突 | DashScope + OpenAI + Anthropic 自动配置可能冲突 | 仅保留 DashScope 自动配置，其他通过 exclude + 手动创建 |
| ChatModel 实例内存开销 | 每个 provider 缓存一个 ChatModel 实例 | ChatModel 本身轻量，主要是 HTTP client；合理数量的 provider 不会有问题 |
| API Key 泄露 | 加密密钥丢失或数据库泄露 | 密钥从环境变量读取，不入代码仓库；数据库访问权限控制 |
| 厂商 API 变更 | Spring AI 版本与厂商 API 不兼容 | 跟进 Spring AI 版本更新；每个厂商独立，一个出问题不影响其他 |
| 旧数据迁移失败 | agent.model 映射不到 model_definition | 完善 fallback 机制，始终可回退到默认 DashScope |

# JavaClaw 工具动态配置 — 设计方案

> 版本：v1.0 | 日期：2026-03-11

---

## 1. 现状分析

### 1.1 当前工具系统

JavaClaw 目前拥有 **6 个内置工具**，全部以 Java 代码硬编码实现：

| 工具名 | 分组 | 描述 |
|--------|------|------|
| `exec` | RUNTIME（运行时） | 执行 Shell 命令 |
| `readFile` | FS（文件系统） | 读取文件内容 |
| `writeFile` | FS（文件系统） | 写入文件内容 |
| `editFile` | FS（文件系统） | 查找替换编辑文件 |
| `webSearch` | WEB（网络） | 使用 SerpAPI 搜索网页 |
| `webFetch` | WEB（网络） | 获取 URL 内容 |

### 1.2 现有技术栈

- **工具定义**：`@Component` + `@Tool` 注解（`javaclaw-agent/.../tools/` 目录）
- **工具注册**：`ToolAutoConfiguration` → `MethodToolCallbackProvider` → `ToolsEngine`
- **工具分组**：`ToolGroup` 枚举（FS / RUNTIME / WEB）—— 硬编码
- **工具预设**：`ToolProfile` 枚举（FULL / CODING / MESSAGING / MINIMAL）—— 硬编码
- **Agent 绑定**：`agent.tools_enabled` 字段存储 profile 名称或 JSON 工具名数组

### 1.3 存在的限制

| 限制 | 说明 |
|------|------|
| 无法新增工具 | 新增工具必须编写 Java 代码、重新编译部署 |
| 无法修改工具属性 | 工具的 description、参数说明写死在注解里，用户无法定制 |
| 无法接入外部服务 | 不支持通过配置接入第三方 API 或 MCP Server |
| 分组/预设不可扩展 | `ToolGroup`/`ToolProfile` 为枚举，页面无法新增分组或预设 |
| 前端功能单一 | 工具设置页面只能开关已有工具，无法创建、编辑、删除 |

## 2. 目标能力

### 2.1 整体目标

将工具系统从 **"代码定义、页面开关"** 升级为 **"代码+配置+远程 三种来源、页面全生命周期管理"**。

### 2.2 三层工具来源

```
┌────────────────────────────────────────────────────────┐
│                    ToolsEngine 统一注册表                 │
│                                                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │   内置工具     │  │  自定义工具    │  │  MCP 远程工具  │  │
│  │  (Java 代码)  │  │  (DB 定义)    │  │  (MCP 协议)   │  │
│  │              │  │              │  │              │  │
│  │ exec         │  │ HTTP API     │  │ MCP Server A │  │
│  │ readFile     │  │ Shell 命令    │  │ MCP Server B │  │
│  │ writeFile    │  │              │  │ ...          │  │
│  │ editFile     │  │ 属性可从 DB   │  │              │  │
│  │ webSearch    │  │ 覆盖         │  │              │  │
│  │ webFetch     │  │              │  │              │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
└────────────────────────────────────────────────────────┘
                          │
                          ▼
                   AgentRuntime 执行
```

**来源 1：内置工具**
- 保留现有 6 个 Java 实现的工具
- 新增能力：从前端覆盖 description、参数说明、超时时间等属性
- 覆盖数据存入 DB，运行时与代码定义合并

**来源 2：自定义工具**
- 用户通过前端 UI 创建全新的工具
- 支持两种执行类型：
  - **HTTP API**：配置 URL、Method、Headers、Body 模板，运行时发起 HTTP 请求
  - **Shell Command**：配置命令模板，运行时执行 Shell 命令
- 工具定义（名称、描述、参数 Schema、执行配置）完整存储在 DB
- 运行时动态生成 `ToolCallback` 实例

**来源 3：MCP Server**
- 用户通过前端 UI 配置外部 MCP (Model Context Protocol) Server 连接
- JavaClaw 作为 MCP Client，通过标准协议自动发现远程 Server 提供的工具
- 支持 SSE、STDIO、Streamable HTTP 三种传输方式
- Spring AI 1.0 原生支持 `AsyncMcpToolCallbackProvider`

### 2.3 能力矩阵

| 能力 | 内置工具 | 自定义工具 | MCP 工具 |
|------|---------|-----------|---------|
| 前端创建 | — | 支持 | 通过配置 MCP Server |
| 前端编辑属性 | 覆盖 description/超时等 | 完整编辑 | 只读（由 Server 定义） |
| 前端删除 | 不可删除 | 支持 | 删除 MCP Server 连接 |
| 每 Agent 开关 | 支持 | 支持 | 支持 |
| 执行方式 | Java 代码 | HTTP / Shell | MCP 协议远程调用 |
| 参数定义 | @ToolParam 注解 | JSON Schema | MCP Server 声明 |
| 热重载 | 不需要 | DB 变更后重载 | 连接变更后重载 |

## 3. 数据模型

### 3.1 `tool_definition` 表

存储自定义工具的完整定义和内置工具的属性覆盖。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | 自增主键 |
| `name` | VARCHAR(128) UNIQUE | 工具名称（唯一标识） |
| `type` | VARCHAR(32) | 类型：`BUILTIN_OVERRIDE` / `HTTP_API` / `SHELL_COMMAND` |
| `description` | TEXT | 工具描述（LLM 可见） |
| `parameters_schema` | JSON | 参数定义（JSON Schema 格式） |
| `execution_config` | JSON | 执行配置（见下方详细说明） |
| `timeout_seconds` | INT | 执行超时（秒） |
| `group_name` | VARCHAR(64) | 所属分组名称 |
| `agent_name` | VARCHAR(128) NULL | 绑定的 Agent（NULL 表示全局可用） |
| `enabled` | TINYINT DEFAULT 1 | 是否启用 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

**`execution_config` 字段结构：**

HTTP API 类型：
```json
{
  "url": "https://api.example.com/translate",
  "method": "POST",
  "headers": {
    "Authorization": "Bearer ${env:API_KEY}",
    "Content-Type": "application/json"
  },
  "bodyTemplate": "{\"text\": \"${text}\", \"targetLang\": \"${targetLang}\"}",
  "responseExtract": "$.result.translation"
}
```

Shell Command 类型：
```json
{
  "commandTemplate": "docker exec ${container} ${command}",
  "workingDirectory": "/workspace",
  "envVars": { "PATH": "/usr/local/bin:/usr/bin" }
}
```

BUILTIN_OVERRIDE 类型：
```json
{
  "builtinName": "exec",
  "overrides": {
    "description": "在安全沙箱中执行 Shell 命令",
    "timeoutSeconds": 60
  }
}
```

### 3.2 `mcp_server` 表

存储外部 MCP Server 连接配置。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | 自增主键 |
| `name` | VARCHAR(128) UNIQUE | 服务名称（显示用） |
| `server_url` | VARCHAR(512) | SSE/HTTP URL 或 STDIO 命令 |
| `transport_type` | VARCHAR(32) | 传输类型：`SSE` / `STDIO` / `STREAMABLE_HTTP` |
| `auth_config` | JSON NULL | 认证配置 |
| `agent_name` | VARCHAR(128) NULL | 绑定的 Agent（NULL 表示全局） |
| `enabled` | TINYINT DEFAULT 1 | 是否启用 |
| `last_connected_at` | DATETIME NULL | 最后成功连接时间 |
| `tools_discovered` | JSON NULL | 上次发现的工具列表（缓存） |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

**`auth_config` 字段结构：**
```json
{
  "type": "bearer",
  "token": "${env:MCP_SERVER_TOKEN}"
}
```

或 Header 认证：
```json
{
  "type": "header",
  "headers": {
    "X-API-Key": "your-api-key"
  }
}
```

STDIO 类型的 `server_url`：
```json
{
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"],
  "env": {}
}
```

## 5. 交互流程

### 5.1 自定义工具创建流程

```
用户点击 [+ 创建新工具]
    │
    ▼
填写工具名称、描述
    │
    ▼
选择执行类型 ──────┐
    │              │
    │ HTTP API     │ Shell Command
    ▼              ▼
填写 URL/Method/   填写命令模板/
Headers/Body       工作目录
    │              │
    ├──────────────┘
    ▼
定义参数（名称、类型、必填、描述）
    │
    ▼
[测试执行] ── 填写测试参数 → 显示执行结果
    │
    ▼
[保存] → 写入 DB → 通知 ToolsEngine 重载
    │
    ▼
新工具出现在内置工具 Tab 的列表中（分组为 CUSTOM）
可以在 Agent 的 toolsEnabled 中开启/关闭
```

### 5.2 MCP Server 配置流程

```
用户点击 [+ 添加服务]
    │
    ▼
填写服务名称、传输类型
    │
    ▼
填写连接信息（URL / 命令）
    │
    ▼
配置认证（可选）
    │
    ▼
[测试连接]
    │
    ├── 失败 → 显示错误信息 → 修改配置重试
    │
    └── 成功 → 显示发现的工具列表
         │
         ▼
    [保存] → 写入 DB → 建立 MCP Client 连接
         │
         ▼
    MCP 工具出现在内置工具 Tab
    分组为 "MCP:服务名称"
    可以在 Agent 的 toolsEnabled 中开启/关闭
```

### 5.3 内置工具属性覆盖流程

```
用户在内置工具列表点击 [编辑]
    │
    ▼
弹出属性编辑面板
    │
    ▼
修改 description / 超时时间
    │
    ▼
[确定] → 写入 tool_definition 表（type=BUILTIN_OVERRIDE）
    │
    ▼
ToolsEngine 重载 → 该工具的 ToolDefinition 使用覆盖值
    │
    ▼
[恢复默认] → 删除 DB 覆盖记录 → 使用 Java 注解原始值
```

## 6. 安全边界

### 6.1 自定义 HTTP 工具

| 风险 | 对策 |
|------|------|
| SSRF（服务端请求伪造） | URL 白名单/黑名单机制，禁止内网 IP（`10.x`、`172.16-31.x`、`192.168.x`、`127.0.0.1`） |
| 敏感信息泄露 | `auth_config` 中的 token 等敏感字段加密存储，API 返回时脱敏 |
| 超时/资源耗尽 | 强制最大超时上限（如 120 秒），限制响应体大小（如 1MB） |
| 注入攻击 | Body 模板参数做 JSON 转义，禁止模板中直接拼接 URL path |

### 6.2 自定义 Shell 命令工具

| 风险 | 对策 |
|------|------|
| 命令注入 | 参数通过 `ProcessBuilder` 的 args 传入，不做字符串拼接 |
| 权限逃逸 | 限制允许的命令前缀白名单（可配置） |
| 资源耗尽 | 强制最大超时，限制输出长度 |
| 文件系统访问 | 限制工作目录在安全范围内 |

### 6.3 MCP Server

| 风险 | 对策 |
|------|------|
| 不受信任的 Server | 连接时需要测试确认；工具列表前端可见，用户明确开启 |
| Token 泄露 | 认证配置加密存储，前端展示脱敏 |
| 连接失败 | 超时机制 + 重试策略 + 状态监控 |
| 工具名冲突 | MCP 工具名加前缀 `mcp_{serverName}_{toolName}`，避免与内置/自定义工具冲突 |

### 6.4 通用安全规则

- 所有工具定义的 CRUD 操作需要鉴权（现有 token 认证机制）
- 工具名称命名规则：`[a-zA-Z][a-zA-Z0-9_]{1,63}`，防止特殊字符注入
- 全局工具数量上限（如 200 个），防止过度创建
- 操作审计：工具定义的创建/修改/删除记录到 `audit_log` 表

## 7. 与现有功能的兼容性

### 7.1 `tools_enabled` 字段兼容

现有的 `agent.tools_enabled` 字段存储格式保持不变：
- `null` / `"full"` → 启用所有工具（包含内置 + 自定义 + MCP）
- profile 名称（`"coding"`）→ 仅匹配内置工具的预设
- JSON 数组 → 精确指定工具名列表（支持任意来源的工具名）

### 7.2 ToolProfile / ToolGroup 过渡

- 现有枚举保留，作为内置工具的"快速预设"
- 自定义工具默认归入 `CUSTOM` 分组
- MCP 工具默认归入 `MCP:{serverName}` 分组
- 分组信息从枚举 + DB 两个来源合并后返回前端

### 7.3 Agent Loop 兼容

`AgentRuntime` 的执行流程不需要修改：
- `ToolsEngine.getToolCallbacksFor()` 返回的 `ToolCallback[]` 已包含所有来源
- `ToolsEngine.getCallbackByName()` 在统一注册表中查找
- 对 `AgentRuntime` 而言，所有工具都是标准的 `ToolCallback`，无需区分来源

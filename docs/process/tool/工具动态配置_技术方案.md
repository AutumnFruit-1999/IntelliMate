```
                           │      AgentRuntime            │
```

### 1.2 核心设计原则

1. **统一接口**：所有工具来源最终产出标准 `ToolCallback`，`AgentRuntime` 无需区分来源
2. **热重载**：DB 工具定义或 MCP 连接变更后，调用 `refresh()` 重建注册表，不需要重启
3. **向后兼容**：现有 `ToolsEngine` 的公开 API（`getToolCallbacksFor`、`getCallbackByName`、`getToolMetadata`）签名保持不变
4. **安全隔离**：自定义工具执行在独立线程池，异常不影响内置工具

## 4. 动态 ToolCallback 实现

### 4.1 HttpToolCallback

基于 `WebClient` 执行 HTTP 调用，实现 Spring AI 的 `ToolCallback` 接口。

```java
package com.atm.intellimate.agent.tools.dynamic;

public class HttpToolCallback implements ToolCallback {

    private final ToolDefinition toolDefinition;
    private final HttpExecutionConfig config;
    private final WebClient webClient;
    private final int timeoutSeconds;

    // 构造函数接收从 DB 加载的配置
    public HttpToolCallback(ToolDefinitionEntity entity, WebClient webClient) {
        this.config = parseConfig(entity.getExecutionConfig());
        this.toolDefinition = buildToolDefinition(entity);
        this.webClient = webClient;
        this.timeoutSeconds = entity.getTimeoutSeconds();
    }

    @Override
    public ToolDefinition getToolDefinition() {
        return toolDefinition;
    }

    @Override
    public String call(String toolInput) {
        // 1. 解析 toolInput (JSON) 为 Map
        Map<String, Object> args = parseArguments(toolInput);

        // 2. 模板变量替换
        String url = resolveTemplate(config.getUrl(), args);
        String body = resolveTemplate(config.getBodyTemplate(), args);
        Map<String, String> headers = resolveHeaders(config.getHeaders(), args);

        // 3. 执行 HTTP 请求
        String response = webClient.method(HttpMethod.valueOf(config.getMethod()))
                .uri(url)
                .headers(h -> headers.forEach(h::set))
                .bodyValue(body)
                .retrieve()
                .bodyToMono(String.class)
                .timeout(Duration.ofSeconds(timeoutSeconds))
                .block();

        // 4. 响应提取（JSONPath）
        if (config.getResponseExtract() != null) {
            return extractByJsonPath(response, config.getResponseExtract());
        }
        return response;
    }

    // 模板解析：将 ${paramName} 替换为实际参数值
    private String resolveTemplate(String template, Map<String, Object> args) {
        if (template == null) return null;
        String result = template;
        for (Map.Entry<String, Object> entry : args.entrySet()) {
            result = result.replace("${" + entry.getKey() + "}", String.valueOf(entry.getValue()));
        }
        // 环境变量解析：${env:VAR_NAME}
        result = ENV_PATTERN.matcher(result).replaceAll(m ->
                System.getenv(m.group(1)) != null ? System.getenv(m.group(1)) : "");
        return result;
    }

    // 从 entity 构建 Spring AI ToolDefinition
    private ToolDefinition buildToolDefinition(ToolDefinitionEntity entity) {
        return ToolDefinition.builder()
                .name(entity.getName())
                .description(entity.getDescription())
                .inputSchema(entity.getParametersSchema())
                .build();
    }
}
```

**核心数据结构 HttpExecutionConfig：**

```java

    String url,
    String method,
    Map<String, String> headers,
    String bodyTemplate,
    String responseExtract
) {}
```

### 4.2 ShellToolCallback

基于 `ProcessBuilder` 执行 Shell 命令。

```java
package com.atm.intellimate.agent.tools.dynamic;

public class ShellToolCallback implements ToolCallback {

    private final ToolDefinition toolDefinition;
    private final ShellExecutionConfig config;
    private final int timeoutSeconds;

    @Override
    public String call(String toolInput) {
        Map<String, Object> args = parseArguments(toolInput);

        // 1. 命令模板替换（注意：参数不做 Shell 拼接，通过 args 传入）
        String command = resolveTemplate(config.commandTemplate(), args);

        // 2. 安全校验
        validateCommand(command);

        // 3. 通过 ProcessBuilder 执行
        ProcessBuilder pb = new ProcessBuilder("sh", "-c", command);
        if (config.workingDirectory() != null) {
            pb.directory(new File(config.workingDirectory()));
        }
        pb.redirectErrorStream(true);
        if (config.envVars() != null) {
            pb.environment().putAll(config.envVars());
        }

        Process process = pb.start();
        boolean finished = process.waitFor(timeoutSeconds, TimeUnit.SECONDS);
        if (!finished) {
            process.destroyForcibly();
            return "ERROR: Command timed out after " + timeoutSeconds + "s";
        }

        String output = new String(process.getInputStream().readAllBytes(), StandardCharsets.UTF_8);
        int exitCode = process.exitValue();
        return "exit_code=" + exitCode + "\n" + output;
    }

    private void validateCommand(String command) {
        // 检查命令白名单/黑名单
        // 禁止危险命令：rm -rf /、shutdown 等
    }
}
```

**核心数据结构 ShellExecutionConfig：**

```java
    String commandTemplate,
    String workingDirectory,
    Map<String, String> envVars
```

### 4.3 BuiltinOverrideApplier

内置工具属性覆盖不创建新的 ToolCallback，而是包装现有 ToolCallback 替换 ToolDefinition：

```java
package com.atm.intellimate.agent.tools.dynamic;

public class BuiltinOverrideApplier {

    /**
     * 在原始 ToolCallback 基础上覆盖 description 等属性。
     * 返回一个代理 ToolCallback，执行逻辑不变，仅 ToolDefinition 被替换。
     */
    public static ToolCallback apply(ToolCallback original, ToolDefinitionEntity override) {
        BuiltinOverrideConfig config = parseConfig(override.getExecutionConfig());

        ToolDefinition originalDef = original.getToolDefinition();
        ToolDefinition.Builder builder = ToolDefinition.builder()
                .name(originalDef.name());

        String desc = config.overrides().get("description");
        builder.description(desc != null ? desc : originalDef.description());

        builder.inputSchema(originalDef.inputSchema());

        ToolDefinition newDef = builder.build();

        return new ToolCallback() {
            @Override
            public ToolDefinition getToolDefinition() { return newDef; }

            @Override
            public String call(String toolInput) { return original.call(toolInput); }
        };
    }
}
```

### 4.4 JSON Schema → ToolDefinition 转换

自定义工具的 `parameters_schema` 字段存储标准 JSON Schema，需转换为 Spring AI 的 `ToolDefinition.inputSchema`：

```java
private ToolDefinition buildToolDefinition(ToolDefinitionEntity entity) {
    // Spring AI ToolDefinition 的 inputSchema 本身就是 JSON Schema 字符串
    // 直接使用 entity.getParametersSchema() 即可
    return ToolDefinition.builder()
            .name(entity.getName())
            .description(entity.getDescription())
            .inputSchema(entity.getParametersSchema())
            .build();
}
```

Spring AI 的 `ToolDefinition.inputSchema` 接受标准 JSON Schema 字符串，与 DB 中的 `parameters_schema` 格式天然兼容，无需额外转换层。

### 5.2 程序化创建 MCP Client

不使用 `application.yml` 静态配置，改为从 DB 配置动态创建：

```java
package com.atm.intellimate.agent.tools.mcp;

@Component
public class McpToolProvider {

    private final Map<String, McpAsyncClient> clients = new ConcurrentHashMap<>();
    private final Map<String, ToolCallback[]> mcpToolCallbacks = new ConcurrentHashMap<>();

    /**
     * 根据 DB 中的 McpServerEntity 创建或重建 MCP Client 连接
     */
    public Mono<Void> connectServer(McpServerEntity server) {
        return disconnectServer(server.getName())
                .then(Mono.defer(() -> {
                    McpAsyncClient client = createClient(server);
                    return client.initialize()
                            .then(Mono.fromCallable(() -> {
                                clients.put(server.getName(), client);
                                // 使用 AsyncMcpToolCallbackProvider 发现工具
                                AsyncMcpToolCallbackProvider provider =
                                        AsyncMcpToolCallbackProvider.builder()
                                                .mcpClients(List.of(client))
                                                .build();
                                ToolCallback[] callbacks = provider.getToolCallbacks();
                                // 工具名加前缀防冲突
                                ToolCallback[] prefixed = prefixToolNames(server.getName(), callbacks);
                                mcpToolCallbacks.put(server.getName(), prefixed);
                                return (Void) null;
                            }));
                }));
    }

    public Mono<Void> disconnectServer(String serverName) {
        McpAsyncClient existing = clients.remove(serverName);
        mcpToolCallbacks.remove(serverName);
        if (existing != null) {
            return existing.closeGracefully();
        }
        return Mono.empty();
    }

    /**
     * 获取所有 MCP Server 提供的 ToolCallback
     */
    public ToolCallback[] getAllCallbacks() {
        return mcpToolCallbacks.values().stream()
                .flatMap(Arrays::stream)
                .toArray(ToolCallback[]::new);
    }

    /**
     * 测试连接并返回发现的工具列表
     */
    public Mono<List<ToolInfo>> testConnection(McpServerEntity server) {
        McpAsyncClient client = createClient(server);
        return client.initialize()
                .then(client.listTools())
                .map(result -> result.tools().stream()
                        .map(t -> new ToolInfo(t.name(), t.description()))
                        .toList())
                .doFinally(signal -> client.closeGracefully().subscribe());
    }

    private McpAsyncClient createClient(McpServerEntity server) {
        return switch (server.getTransportType()) {
            case "SSE" -> McpClient.async(
                    WebFluxSseClientTransport.builder(server.getServerUrl()).build()
            ).build();

            case "STREAMABLE_HTTP" -> McpClient.async(
                    WebFluxStreamableHttpClientTransport.builder(server.getServerUrl()).build()
            ).build();

            case "STDIO" -> {
                StdioConfig config = parseStdioConfig(server.getServerUrl());
                yield McpClient.async(
                        StdioClientTransport.builder(config.command())
                                .args(config.args())
                                .build()
                ).build();
            }

            default -> throw new IllegalArgumentException(
                    "Unsupported transport: " + server.getTransportType());
        };
    }

    /**
     * 为 MCP 工具名添加前缀，避免与内置/自定义工具冲突
     * 例：readFile → mcp_filesystem_readFile
     */
    private ToolCallback[] prefixToolNames(String serverName, ToolCallback[] callbacks) {
        String prefix = "mcp_" + serverName.replaceAll("[^a-zA-Z0-9]", "_") + "_";
        return Arrays.stream(callbacks)
                .map(cb -> new PrefixedToolCallback(prefix, cb))
                .toArray(ToolCallback[]::new);
    }
}
```

### 5.3 MCP 连接生命周期

```
应用启动
    │
    ▼
McpToolProvider.init()
    │
    ▼
从 DB 加载所有 enabled 的 McpServerEntity
    │
    ▼
逐个调用 connectServer() ─────┐
    │                          │ 失败 → 记录日志，标记状态
    │                          │ 定时重试（30s 间隔）
    ├── 成功 → 缓存 callbacks  │
    │                          │
    ▼                          │
ToolCallback[] 就绪            │
                               │
API 添加/修改 MCP Server ──────┘
    │
    ▼
connectServer() → 重新发现工具 → 通知 CompositeToolsEngine.refresh()
```

### 5.4 工具名冲突处理策略

| 来源 | 工具名格式 | 示例 |
|------|-----------|------|
| 内置 | 原始方法名 | `exec`、`readFile` |
| 自定义 | 用户定义名称 | `translate_text`、`docker_exec` |
| MCP | `mcp_{serverName}_{toolName}` | `mcp_filesystem_readFile` |

创建自定义工具时校验名称不与内置工具冲突。MCP 工具通过前缀强制避免冲突。

## 6. CompositeToolsEngine 重构

### 6.1 改造方案

将现有 `ToolsEngine` 重构为 `CompositeToolsEngine`，管理三个工具 Provider：

```java
package com.atm.intellimate.agent.tools;

@Component
public class CompositeToolsEngine {

    private final BuiltinToolProvider builtinProvider;
    private final DynamicToolProvider dynamicProvider;
    private final McpToolProvider mcpProvider;

    private volatile ToolCallback[] allToolCallbacks;

    @PostConstruct
    public void init() {
        refresh();
    }

    /**
     * 重建统一注册表。
     * 在以下时机调用：
     *   - 应用启动
     *   - 自定义工具 CRUD 后
     *   - MCP Server 连接变更后
     */
    public synchronized void refresh() {
        List<ToolCallback> all = new ArrayList<>();

        // 1. 内置工具
        ToolCallback[] builtins = builtinProvider.getToolCallbacks();

        // 2. 从 DB 加载内置覆盖，应用到 builtins
        Map<String, ToolDefinitionEntity> overrides = dynamicProvider.getBuiltinOverrides();
        for (ToolCallback cb : builtins) {
            String name = cb.getToolDefinition().name();
            ToolDefinitionEntity override = overrides.get(name);
            if (override != null) {
                all.add(BuiltinOverrideApplier.apply(cb, override));
            } else {
                all.add(cb);
            }
        }

        // 3. 自定义工具（HTTP_API / SHELL_COMMAND）
        all.addAll(dynamicProvider.getDynamicCallbacks());

        // 4. MCP 工具
        ToolCallback[] mcpCallbacks = mcpProvider.getAllCallbacks();
        all.addAll(Arrays.asList(mcpCallbacks));

        this.allToolCallbacks = all.toArray(new ToolCallback[0]);

        log.info("CompositeToolsEngine refreshed: {} builtin, {} custom, {} mcp, {} total",
                builtins.length,
                dynamicProvider.getDynamicCallbacks().size(),
                mcpCallbacks.length,
                allToolCallbacks.length);
    }

    // ─── 公开 API（签名与现有 ToolsEngine 保持一致）───

    public ToolCallback[] getToolCallbacks() {
        return allToolCallbacks;
    }

    public ToolCallback[] getToolCallbacksFor(String toolsEnabledSpec) {
        // 与现有逻辑相同：null/full → 全部，profile → 预设，JSON array → 精确匹配
        // ...
    }

    public ToolCallback getCallbackByName(String toolName) {
        return Arrays.stream(allToolCallbacks)
                .filter(cb -> cb.getToolDefinition().name().equals(toolName))
                .findFirst()
                .orElseThrow(() -> new IllegalArgumentException("Unknown tool: " + toolName));
    }

    public List<Map<String, String>> getToolMetadata() {
        return Arrays.stream(allToolCallbacks).map(cb -> {
            String name = cb.getToolDefinition().name();
            String desc = cb.getToolDefinition().description();
            String source = detectSource(name); // "builtin" / "custom" / "mcp"
            String group = resolveGroup(name);
            String groupDisplayName = resolveGroupDisplayName(name);

            Map<String, String> meta = new LinkedHashMap<>();
            meta.put("name", name);
            meta.put("description", desc != null ? desc : "");
            meta.put("source", source);
            meta.put("group", group);
            meta.put("groupDisplayName", groupDisplayName);
            return meta;
        }).toList();
    }

    /**
     * 返回所有分组（合并枚举 + CUSTOM + MCP:{name}）
     */
    public List<Map<String, Object>> getAllGroups() {
        List<Map<String, Object>> groups = new ArrayList<>();

        // 内置分组（从 ToolGroup 枚举）
        for (ToolGroup g : ToolGroup.values()) {
            groups.add(Map.of(
                    "name", g.name(),
                    "displayName", g.getDisplayName(),
                    "tools", List.copyOf(g.getToolNames()),
                    "source", "builtin"
            ));
        }

        // CUSTOM 分组（所有自定义工具）
        List<String> customToolNames = dynamicProvider.getDynamicCallbacks().stream()
                .map(cb -> cb.getToolDefinition().name())
                .toList();
        if (!customToolNames.isEmpty()) {
            groups.add(Map.of(
                    "name", "CUSTOM",
                    "displayName", "自定义工具",
                    "tools", customToolNames,
                    "source", "custom"
            ));
        }

        // MCP 分组（每个 MCP Server 一个分组）
        mcpProvider.getServerToolNames().forEach((serverName, toolNames) -> {
            groups.add(Map.of(
                    "name", "MCP:" + serverName,
                    "displayName", "MCP: " + serverName,
                    "tools", toolNames,
                    "source", "mcp"
            ));
        });

        return groups;
    }
```

### 6.2 DynamicToolProvider

从 DB 加载自定义工具定义并生成 `ToolCallback`：

```java
@Component
public class DynamicToolProvider {

    private final ToolDefinitionRepository repository;
    private final WebClient webClient;
    private volatile List<ToolCallback> dynamicCallbacks = List.of();
    private volatile Map<String, ToolDefinitionEntity> builtinOverrides = Map.of();

    /**
     * 从 DB 重新加载所有工具定义
     */
    public Mono<Void> reload() {
        return repository.findAllByEnabled(1)
                .collectList()
                .doOnNext(entities -> {
                    List<ToolCallback> callbacks = new ArrayList<>();
                    Map<String, ToolDefinitionEntity> overrides = new HashMap<>();

                    for (ToolDefinitionEntity entity : entities) {
                        switch (entity.getType()) {
                            case "HTTP_API" ->
                                callbacks.add(new HttpToolCallback(entity, webClient));
                            case "SHELL_COMMAND" ->
                                callbacks.add(new ShellToolCallback(entity));
                            case "BUILTIN_OVERRIDE" ->
                                overrides.put(extractBuiltinName(entity), entity);
                        }
                    }

                    this.dynamicCallbacks = List.copyOf(callbacks);
                    this.builtinOverrides = Map.copyOf(overrides);
                })
                .then();
    }

    public List<ToolCallback> getDynamicCallbacks() { return dynamicCallbacks; }
    public Map<String, ToolDefinitionEntity> getBuiltinOverrides() { return builtinOverrides; }
}
```

## 7. 热重载机制

### 7.1 触发时机

| 操作 | 触发方式 |
|------|---------|
| 创建/更新/删除自定义工具 | Controller 方法完成后调用 `dynamicProvider.reload().then(compositeToolsEngine.refresh())` |
| 添加/修改/删除 MCP Server | Controller 方法完成后调用 `mcpProvider.connectServer()`/`disconnectServer()` → `compositeToolsEngine.refresh()` |
| 应用启动 | `@PostConstruct` → `dynamicProvider.reload()` → `mcpProvider.init()` → `compositeToolsEngine.refresh()` |

### 7.2 线程安全

- `CompositeToolsEngine.allToolCallbacks` 使用 `volatile` 保证可见性
- `refresh()` 使用 `synchronized` 防止并发重建
- `AgentRuntime` 读取 `allToolCallbacks` 是无锁的，只在调用 `getToolCallbacksFor()` 时获取当前快照

### 7.3 重载流程

```
CRUD API 完成
      │
      ▼
DynamicToolProvider.reload()  ─── 从 DB 重新加载所有定义
      │
      ▼
CompositeToolsEngine.refresh()  ─── 重建 allToolCallbacks[]
      │
      ▼
后续的 AgentRuntime.dispatch() 自动使用新工具集
```

### 11.1 URL 安全校验（HTTP 工具）

```java
public class UrlValidator {
    private static final Set<String> BLOCKED_HOSTS = Set.of(
        "localhost", "127.0.0.1", "0.0.0.0", "::1"
    );
    private static final List<String> BLOCKED_CIDR = List.of(
        "10.", "172.16.", "172.17.", /* ... */ "172.31.", "192.168."
    );

    public static void validate(String url) {
        URI uri = URI.create(url);
        String host = uri.getHost();
        if (BLOCKED_HOSTS.contains(host)) {
            throw new SecurityException("Blocked host: " + host);
        }
        // 解析 IP 并检查 CIDR
        InetAddress addr = InetAddress.getByName(host);
        String ip = addr.getHostAddress();
        for (String prefix : BLOCKED_CIDR) {
            if (ip.startsWith(prefix)) {
                throw new SecurityException("Blocked internal IP: " + ip);
            }
        }
    }
}
```

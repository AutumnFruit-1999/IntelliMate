# 多智能体系统设计文档

## 1. 功能概述

JavaClaw 需要支持创建和管理多个独立的智能体（Agent），每个智能体拥有独立的：
- 名称和模型配置
- SOUL / USER / AGENTS 上下文文件
- 工具权限（tools_enabled）
- 系统提示词（system_prompt）
- 会话历史（每个 agent 的会话相互隔离）

用户可以在前端界面中自由切换不同的智能体进行对话。

### 2.4 智能体切换行为

- 切换智能体时，ChatPanel 清空当前消息显示
- 加载目标智能体的会话历史（如有）
- Sidebar 的"智能体配置"区域自动关联到当前智能体
- WebSocket 连接保持不变，通过请求参数切换 agent

### 2.5 智能体配置面板

与已有设计一致（SOUL / USER / AGENTS 三个 Tab），但关联到当前选中的智能体。

## 3. 数据模型

### 3.1 实体关系

```
Agent (1) ──── (N) Session (1) ──── (N) TranscriptMessage
  │
  ├── name (唯一标识)
  ├── model
  ├── system_prompt
  ├── soul_md / user_md / agents_md
  └── tools_enabled
```

### 3.2 会话隔离策略

每个智能体 + 每个用户（contextId）形成独立的会话：

```
SessionKey = channelId + contextType + contextId
```

当用户切换智能体时，`contextId` 应包含 agent 信息以实现会话隔离：

```
contextId = wsSessionId + ":" + agentName
```

这样同一用户对不同智能体的对话历史互不干扰。

## 4. API 设计

### 4.1 智能体 CRUD

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/agents` | 列出所有智能体 |
| GET | `/api/agent/{name}` | 获取单个智能体详情 |
| POST | `/api/agent` | 创建新智能体 |
| PUT | `/api/agent/{name}` | 更新智能体配置 |
| PUT | `/api/agent/{name}/context` | 更新上下文文件（已有） |
| DELETE | `/api/agent/{name}` | 删除智能体（软删除） |

### 4.2 WebSocket 协议扩展

`conversation.message` 请求新增 `agentName` 参数：

```json
{
  "type": "request",
  "id": "uuid",
  "method": "conversation.message",
  "params": {
    "text": "你好",
    "channelId": "webchat",
    "contextType": "dm",
    "agentName": "coder"
  }
}
```

- `agentName` 不为空时：使用指定智能体，`contextId` 自动加上 agent 前缀
- `agentName` 为空时：使用默认智能体（向后兼容）

### 4.3 会话欢迎事件增强

```json
{
  "type": "event",
  "event": "session.welcome",
  "payload": {
    "wsSessionId": "abc123",
    "defaultAgent": "javaclaw"
  }
}
```

## 5. 交互流程

### 5.1 首次进入

```
用户打开页面
  ↓
WebSocket 连接 → session.welcome
  ↓
前端 GET /api/agents → 获取智能体列表
  ↓
自动选中默认智能体（第一个或上次使用的）
  ↓
显示该智能体的会话
```

### 5.2 切换智能体

```
用户点击 Sidebar 中另一个智能体
  ↓
前端更新 activeAgent 状态
  ↓
清空当前消息 / 或加载该 agent 的历史消息
  ↓
后续消息请求携带新的 agentName
  ↓
后端为新 agent 创建/复用独立会话
```

### 5.3 创建新智能体

```
用户点击 "+ 新建智能体"
  ↓
填写名称和模型 → POST /api/agent
  ↓
智能体列表刷新，自动切换到新智能体
  ↓
（可选）自动打开 SOUL 配置面板
```

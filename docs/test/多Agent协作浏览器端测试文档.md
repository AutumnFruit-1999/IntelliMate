# 多 Agent 协作 — 浏览器端测试文档

## 测试环境

- **后端地址**：http://localhost:3007
- **前端地址**：http://localhost:5173（开发模式）
- **浏览器**：Chrome / Edge / Firefox（最新版本）
- **开发者工具**：打开 Console 面板查看 WebSocket 事件日志

---

## 1. 委派配置面板测试

### 1.1 打开委派配置

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 点击页面右上角 Agent 名称 | 弹出 Agent 列表 |
| 2 | 选择任意 Agent（如 `deepseek`）| 切换到该 Agent |
| 3 | 点击 ⚙️ 图标打开 Agent 配置 Modal | 显示配置弹窗 |
| 4 | 切换到 **"委派协作"** Tab | 显示委派配置面板 |

### 1.2 启用委派功能

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 勾选"允许委派任务"复选框 | 出现"可委派的目标 Agent"列表 |
| 2 | 在列表中勾选一个或多个 Agent | 被选中的 Agent 显示蓝色勾选 |
| 3 | 输入"Agent 目标描述" | 文本框接受输入 |
| 4 | 点击"保存委派配置"按钮 | 按钮变为"保存中..."→ 完成后消失 |
| 5 | 关闭弹窗，重新打开委派配置 | 之前保存的配置仍然存在 |

### 1.3 禁用委派功能

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 取消勾选"允许委派任务" | "可委派的目标 Agent"列表隐藏 |
| 2 | 保存配置 | 成功保存 |
| 3 | 发送消息测试 | Agent 不会使用委派工具 |

### 1.4 Agent 列表展示

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 在 Agent 列表页面 | 已启用委派的 Agent 显示 `canDelegate=true` |
| 2 | 查看 Agent 目标描述 | 有 `goal` 的 Agent 显示目标文字 |

## 2. 委派 (Delegation) 消息可视化测试

### 2.1 前置条件

1. 配置 Agent A 启用委派，目标包含 Agent B
2. Agent B 有正确的 goal 描述和可用的模型

### 2.2 触发委派

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 切换到 Agent A | 聊天面板切换到 Agent A |
| 2 | 输入："请委派给 {AgentB名称} 来完成 XXX 任务" | 消息发送 |
| 3 | 等待 Agent A 推理 | 显示 Turn 指示器 |
| 4 | Agent A 调用 `delegateAgent` 工具 | 工具调用卡片显示在消息中 |

### 2.3 委派卡片显示

当 `workflow.delegation_start` 事件到达时：

| 检查项 | 预期表现 |
|--------|----------|
| 卡片位置 | 出现在 Agent 消息气泡上方 |
| 卡片标题 | "⏳ 委派: {Worker名称}" |
| 任务描述 | 显示委派的任务文字 |
| 状态颜色 | 蓝色边框 + 蓝色背景（running 状态）|
| 折叠箭头 | 显示 ▼（可展开）|

### 2.4 委派进度

当 `workflow.delegation_progress` 事件到达时：

| 检查项 | 预期表现 |
|--------|----------|
| Worker 工具调用 | 展开后显示每个工具调用名称和参数 |
| 工具状态图标 | 进行中显示 "…"，成功 "✓"，失败 "✗" |

### 2.5 委派完成

当 `workflow.delegation_result` 事件到达时：

| 检查项 | 预期表现 |
|--------|----------|
| 状态颜色 | 成功：绿色边框/背景；失败：红色边框/背景 |
| 状态图标 | 成功："✓ 委派: {名称}"；失败："✗ 委派: {名称}" |
| 耗时 | 右侧显示 "X.Xs" |
| Turns | 右侧显示 "N turns" |
| 结果预览 | 展开后显示 Worker 的回复内容（最多 500 字符）|

## 3. 移交 (Handoff) 可视化测试

### 3.1 触发移交

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 发送需要移交的消息 | Agent A 判断需要移交给 Agent B |
| 2 | Agent A 调用 `handoffToAgent` | 工具调用显示在消息中 |

### 3.2 移交指示器显示

当 `workflow.handoff` 事件到达时：

| 检查项 | 预期表现 |
|--------|----------|
| 指示器样式 | 琥珀色背景 + 琥珀色边框 |
| 方向箭头 | "↗" 图标 |
| Agent 名称 | "AgentA → AgentB" 格式 |
| 移交原因 | 灰色文字显示原因 |

### 3.3 目标 Agent 接管

| 检查项 | 预期表现 |
|--------|----------|
| 后续消息 | 由目标 Agent 继续回复 |
| 上下文 | 目标 Agent 的回复应体现对之前对话的理解 |

## 4. 并行 (Parallel) 可视化测试

### 4.1 前置条件

Supervisor Agent 的 `delegateAgents` 包含至少 2 个可用 Worker Agent。

### 4.2 触发并行委派

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 发送需要多 Agent 并行的消息 | Agent 调用 `delegateAgentsParallel` |
| 2 | 等待并行开始事件 | 显示并行组卡片 |

### 4.3 并行组显示

当 `workflow.parallel_start` 事件到达时：

| 检查项 | 预期表现 |
|--------|----------|
| 卡片标题 | "⫘ 并行执行 (0/N 完成)" |
| 任务列表 | 每个 Worker 一行，显示名称 + 任务 |
| 状态指示 | 蓝色脉动圆点（running）|
| 整体颜色 | 蓝色边框/背景 |

### 4.4 并行进度

各 Worker 完成时：

| 检查项 | 预期表现 |
|--------|----------|
| 完成进度 | 标题更新为 "⫘ 并行执行 (M/N 完成)" |
| 状态圆点 | 绿色（完成）/ 红色（失败）|
| 耗时 | 每行右侧显示 "X.Xs" |

### 4.5 全部完成

| 检查项 | 预期表现 |
|--------|----------|
| 整体颜色 | 变为绿色边框/背景 |
| 进度 | "⫘ 并行执行 (N/N 完成)" |

## 5. 工作流时间线测试

### 5.1 多步骤工作流

当一条消息中包含多个工作流操作时：

| 检查项 | 预期表现 |
|--------|----------|
| 渲染顺序 | 按事件到达顺序从上到下排列 |
| 混合类型 | 委派卡片、并行组、移交指示器可混合显示 |
| 位置 | 在工具调用组和消息气泡之间 |

### 5.2 复杂场景

- [ ] 先委派一个 Agent，然后并行委派两个 Agent → 三个工作流条目
- [ ] 委派失败后再次委派 → 两张委派卡片（一红一绿）
- [ ] 委派后移交 → 委派卡片 + 移交指示器

## 7. 开发者工具验证

### 7.1 Console 日志

在浏览器 Console 中，可以看到以 `[WS]` 开头的事件日志。新增的事件类型应正确打印：

```
[WS] received plan event: workflow.delegation_start {...}
```

### 7.2 Network 面板

WebSocket 连接的 Messages 标签页中：

| 方向 | 检查项 |
|------|--------|
| ↑ 发送 | `conversation.message` 请求格式正确 |
| ↓ 接收 | `workflow.*` 事件包含正确的 payload 字段 |

### 7.3 Components 面板 (React DevTools)

| 组件 | 检查项 |
|------|--------|
| `WorkflowTimeline` | `entries` prop 包含正确的工作流条目 |
| `DelegationCard` | `delegation` prop 状态正确更新 |
| `ParallelGroup` | `agentStates` 随进度事件更新 |
| `HandoffIndicator` | `info` prop 包含 fromAgent/toAgent |

## 8. 边界场景测试

| 场景 | 操作 | 预期结果 |
|------|------|----------|
| 无可委派 Agent | 委派配置中不勾选任何 Agent | Agent 不会尝试委派 |
| 自委派 | 在委派配置中不显示自己 | Agent 列表自动排除当前 Agent |
| 快速连续消息 | 在委派进行中发送新消息 | isWaiting 状态阻止重复发送 |
| 网络断开 | 委派过程中断网 | WebSocket 重连后状态恢复 |
| 超长 Worker 结果 | Worker 返回 > 500 字符 | 结果预览显示 "..." 截断 |
| Agent 不存在 | 委派给已删除的 Agent | 委派失败，显示红色卡片 |
